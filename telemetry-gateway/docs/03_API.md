## 1. REST API Specs

All REST endpoints serve and accept JSON payloads. Standard HTTP status codes are used for response status.

### 1.1 Source Registration
- **Endpoint**: `POST /gateway/register-source`
- **Description**: Registers an external telemetry source with the Gateway.
- **Request Body**:
```json
{
  "source_type": "REPLAY",
  "source_name": "replay-sim-01",
  "mission": {
    "mission_id": 1,
    "mission_name": "Chandrayaan-3",
    "mission_code": "CY3"
  },
  "satellite": {
    "satellite_id": 101,
    "satellite_name": "Propulsion Module",
    "norad_id": 57479
  },
  "station": {
    "station_id": 401,
    "station_name": "ISTRAC-Bangalore",
    "station_code": "ISTRAC-BLR"
  }
}
```
- **Responses**:
  - `201 Created`: Source registered successfully.
    ```json
    {
      "source_id": "8c4fb2d2-8f9f-4318-8f55-7d52aeb81765",
      "registered_at": "2026-07-03T12:30:29Z",
      "status": "REGISTERED"
    }
    ```
  - `400 Bad Request`: Invalid payload or missing fields.
  - `409 Conflict`: Source with the same name or active configuration already exists.

### 1.2 Unregister Source
- **Endpoint**: `POST /gateway/unregister-source`
- **Description**: Unregisters a telemetry source, freeing its configured properties.
- **Request Body**:
```json
{
  "source_id": "8c4fb2d2-8f9f-4318-8f55-7d52aeb81765"
}
```
- **Responses**:
  - `200 OK`: Source successfully unregistered.
    ```json
    {
      "message": "Source unregistered successfully"
    }
    ```
  - `400 Bad Request`: Source ID missing or source is currently active in a session (must stop session first).
  - `404 Not Found`: Source ID does not exist.

### 1.3 Gateway Status
- **Endpoint**: `GET /gateway/status`
- **Description**: Returns the overall operational status of the Gateway.
- **Responses**:
  - `200 OK`:
    ```json
    {
      "status": "HEALTHY",
      "uptime_seconds": 172800,
      "active_sessions": 2,
      "active_sources": 2,
      "rabbitmq_connected": true,
      "buffer_utilization_pct": 14.5
    }
    ```

### 1.4 Gateway Statistics
- **Endpoint**: `GET /gateway/statistics`
- **Description**: Returns global statistics across all historical and current sessions.
- **Responses**:
  - `200 OK`:
    ```json
    {
      "total_packets_received": 14502930,
      "total_packets_published": 14502900,
      "total_packets_rejected": 30,
      "average_latency_ms": 1.24,
      "peak_throughput_pps": 89000,
      "active_throughput_pps": 3000
    }
    ```

### 1.5 List Sessions
- **Endpoint**: `GET /gateway/sessions`
- **Description**: Returns all currently active telemetry sessions.
- **Responses**:
  - `200 OK`:
    ```json
    [
      {
        "session_id": "f5b6a7c8-d9e0-4a3b-b2c1-d4e5f6a7b8c9",
        "source_id": "8c4fb2d2-8f9f-4318-8f55-7d52aeb81765",
        "started_at": "2026-07-03T12:35:00Z",
        "status": "ACTIVE",
        "packets_received": 150000,
        "packets_published": 150000,
        "packets_rejected": 0,
        "last_packet_at": "2026-07-03T12:40:15Z"
      }
    ]
    ```

### 1.6 Stop Session
- **Endpoint**: `POST /gateway/stop-session`
- **Description**: Forces the termination of an active telemetry session.
- **Request Body**:
```json
{
  "session_id": "f5b6a7c8-d9e0-4a3b-b2c1-d4e5f6a7b8c9"
}
```
- **Responses**:
  - `200 OK`: Session terminated.
    ```json
    {
      "message": "Session termination initiated"
    }
    ```
  - `404 Not Found`: Session ID does not exist or is already inactive.

## 2. gRPC Ingestion API Specs

The hot path for high-throughput streaming utilizes gRPC with HTTP/2 multiplexing.

### 2.1 Service Definition (`ingestion.proto`)

```protobuf
syntax = "proto3";
package must.gateway.v1;

import "must/telemetry/v1/envelope.proto";

service TelemetryIngressService {
  // StreamTelemetry establishes a unidirectional stream for transmitting packets.
  rpc StreamTelemetry(stream TelemetryStreamRequest) returns (TelemetryStreamResponse);
}

message TelemetryStreamRequest {
  string source_id = 1;                             // Registered source ID
  string session_id = 2;                            // Unique session ID generated by source
  must.telemetry.v1.TelemetryEnvelope envelope = 3; // Enveloped packet data
}

message TelemetryStreamResponse {
  string session_id = 1;
  uint64 packets_processed = 2;
  uint64 packets_rejected = 3;
  bool is_healthy = 4;
}
```

## 3. WebSocket API Specs

Used solely for read-only real-time monitoring of status.

- **Endpoint**: `/gateway/ws/status`
- **Format**: JSON Frame
- **Message Types**:
  - **Connection Welcome**: Sent immediately upon connection.
    ```json
    {
      "event": "CONNECTED",
      "server_time": "2026-07-03T12:30:00Z"
    }
    ```
  - **Metrics Tick** (Interval: 1 second):
    ```json
    {
      "event": "METRICS_TICK",
      "data": {
        "active_throughput_pps": 3150,
        "buffer_utilization_pct": 4.2,
        "sessions": [
          {
            "session_id": "f5b6a7c8-d9e0-4a3b-b2c1-d4e5f6a7b8c9",
            "pps": 3150,
            "packets_received": 153150,
            "packets_rejected": 0
          }
        ]
      }
    }
    ```
  - **Alert Event**: Streamed instantly on anomalies.
    ```json
    {
      "event": "ALERT",
      "severity": "CRITICAL",
      "message": "RabbitMQ connection lost. Starting buffering.",
      "timestamp": "2026-07-03T12:36:12Z"
    }
    ```

## 4. Error Registry

All API errors return a standardized error block:
```json
{
  "error_code": "ERR_SOURCE_ALREADY_EXISTS",
  "message": "A source with the name 'replay-sim-01' is already registered.",
  "timestamp": "2026-07-03T12:30:30Z",
  "details": {
    "conflicting_id": "8c4fb2d2-8f9f-4318-8f55-7d52aeb81765"
  }
}
```

| Error Code | HTTP Status | Description |
|------------|-------------|-------------|
| `ERR_INVALID_PAYLOAD` | 400 | The request structure is malformed. |
| `ERR_SOURCE_NOT_FOUND` | 404 | The specified Source ID is not registered. |
| `ERR_SOURCE_ALREADY_EXISTS` | 409 | Conflict in source configurations. |
| `ERR_SESSION_NOT_FOUND` | 404 | The requested Session ID is not active or doesn't exist. |
| `ERR_SESSION_ACTIVE` | 400 | Action failed because the source has an active session. |
| `ERR_BUS_DISCONNECTED` | 503 | RabbitMQ is unavailable and gateway buffer is saturated. |
| `ERR_INTERNAL` | 500 | Unhandled internal error. |
